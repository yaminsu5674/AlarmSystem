# 나의 알림 봇 (디스코드)

GitHub Actions(무료 서버리스)가 정해진 시간에 파이썬을 돌려 **디스코드 웹훅**으로 알림을 보낸다.
서버·과금·앱설치 없음. 나+여친 둘 다 디스코드 채널에서 알림 받음.

## 📦 모듈 4개
| 모듈 | 파일 | 언제 | 하는 일 |
|---|---|---|---|
| 1. 습관 알림 | `habit.py` | 평일 9~18시(12시 제외) 매시 | 눈운동 + 목 스트레칭 가이드 |
| 2. 좌석 조회 | `check_trains.py` | (보류, 수동) | 특정 표 잔여석 뜨면 알림 |
| 3. 예매 오픈 리마인더 | `booking_reminder.py` | 매일 19시 | "내일 7시에 O월 O일 표 열림" |
| 4. 공지 감시 | `notice_watch.py` | 하루 3회 | 코레일·SRT 새 공지(명절 등) 링크 |

## 🚀 세팅 (윈도우 브라우저로 전부 가능)

### 1. 디스코드 웹훅 만들기
디스코드 서버 → 채널 설정(톱니) → **연동 → 웹후크 → 새 웹후크 → URL 복사**
(여친도 이 서버에 초대하면 같은 알림 받음)

### 2. GitHub repo에 올리기
- github.com → 새 repo 생성 (**Private 추천**)
- 이 폴더(`알람어플`) 파일 전부 업로드 (웹에서 드래그)

### 3. Secrets 넣기
repo → **Settings → Secrets and variables → Actions → Secrets 탭**

| 이름 | 값 | 필요 모듈 |
|---|---|---|
| `DISCORD_WEBHOOK` | 1번 URL | 전부 (필수) |
| `KORAIL_ID` / `KORAIL_PW` | 코레일 로그인 | 2번만 |
| `SRT_ID` / `SRT_PW` | SRT 로그인 | 2번만 |

> 습관·예매리마인더·공지감시(1·3·4)는 **웹훅만 있으면 됨.** 기차 로그인은 좌석조회(2) 쓸 때만.

### 4. 켜기 & 테스트
- repo → **Actions 탭** → 워크플로우 활성화(Enable)
- 각 워크플로우 → **Run workflow** 버튼으로 즉시 테스트 → 디스코드 오는지 확인
- 이후 자동 실행

## ⚙️ 커스터마이징
- **습관 문구/시간**: `habit.py` 의 `EYE`·`NECK` 목록, `habit.yml` 의 cron
- **예매 규칙**: 코레일 1개월 전 / SRT 30일 전, 둘 다 07:00 오픈 (2026-07 검증). 바뀌면 `booking_reminder.py` 수정
- **좌석조회 노선**: `routes.json` (2번 켤 때)

## ⚠️ 알아둘 것
- **크론 타이밍**: GitHub Actions는 몇 분 밀리거나 가끔 스킵됨. 예매 리마인더를 **전날 저녁**에 주는 이유(당일 7시 직전은 못 믿음).
- **60일 무활동 시** 스케줄 자동 정지 → 가끔 아무 커밋이나 한 번.
- **공지 감시(4번)**: SRT는 잘 긁힘. **코레일(info.korail.com)은 사이트 구조상 첫 실전 실행 때 안 잡힐 수 있음** → 그러면 셀렉터 손봐야 함(로그 보고 알려줘).
- **좌석조회(2번)**: `korail2`가 코레일 개편으로 가끔 깨짐 → 유지보수 포크로 교체 가능.
- 모든 기차 예매는 **알림만**. 자동예매(매크로)는 약관위반이라 안 함.
